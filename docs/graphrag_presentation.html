04 · Cypher Guard

Cypher 가드 (가장 중요한 안전장치)

LLM이 만든 쿼리는 믿고 바로 돌리지 않아요. 네 단계를 통과해야 DB에 닿습니다. cypher_guard.py:67

✕
쓰기·관리 명령 차단 — CREATE / MERGE / DELETE / SET / REMOVE / DROP / DETACH / LOAD CSV 와 CALL apoc.create·db.create·dbms.* 류는 즉시 거부
⊟
화이트리스트 검사 — 등록된 라벨·관계만 허용. 처음 보는 라벨이 끼면 거부
+
LIMIT 자동 주입 — 쿼리에 LIMIT이 없으면 기본 100을 강제로 붙임
⚡
EXPLAIN dry-run — 실제 실행 없이 Neo4j한테 스키마·문법만 검증시킴. 비용 0, 파라미터까지 바인딩해서 검사
개발자 상세 — 실제 가드 코드 흐름 
def validate_cypher(cypher, session=None, params=None) -> str:
    # ① 쓰기/관리 키워드 정규식 매칭 시 GuardError
    if _FORBIDDEN.search(cypher): raise GuardError("write/admin operation detected")

    # ② 사용된 :Label / :REL 추출 → 화이트리스트 차집합
    used = set(_LABEL_RE.findall(cypher))
    unknown = used - ALLOWED_LABELS - ALLOWED_RELS
    if unknown: raise GuardError(f"unknown labels/rels: {unknown}")

    # ③ LIMIT 없으면 기본 100 주입
    normalized = cypher.strip().rstrip(";")
    if not _LIMIT_RE.search(normalized): normalized += f"\nLIMIT {DEFAULT_LIMIT}"

    # ④ EXPLAIN으로 플랜만 생성(실행 X). 실패 시 GuardError
    if session: session.run("EXPLAIN " + normalized, **params).consume()
    return normalized

WHERE NONE(pat IN ['신탁','자사주','투자조합','사모투자','ETF','KODEX','KoAct']

자회사 재무, 지분경로 (3단계) — GraphRAG의 진짜 가치가 나오는 경로

MATCH (root:Organization {corp_code: $corp_code})
MATCH path = (root)-[:IS_MAJOR_SHAREHOLDER_OF|INVESTS_IN*1..2]->(target:Organization)
WHERE root.corp_code <> target.corp_code
  AND ALL(rr IN relationships(path) WHERE rr.valid_to IS NULL)
MATCH (target)-[:HAS_METRIC]->(m:FinMetric)
WHERE m.account_id = $account_id AND m.reprt_code = '11011' AND m.fs_div = 'CFS'
RETURN DISTINCT target.name AS org, m.bsns_year AS year, m.value AS value
ORDER BY year DESC, value DESC LIMIT 100

07 · Score

Score — 4축 분해 가중치

v2에서 감으로 박아뒀던 단일 점수 테이블을 버리고, "왜 이 점수가 나왔는지" 설명되는 축 곱으로 다시 짰어요. Bloomberg KG의 source-hierarchy 방식을 참고했습니다. score.py

score = source_authority × extraction_method × confidence × time_decay × consistency × intent_boost

source_authority — 출처 권위

DART 정형1.00
비정형0.85

extraction_method — 추출 방식

결정론 적재0.95
멀티홉0.90
LLM 추출0.90
룰 파생0.85
LLM Cypher0.82

fact별 동적 보정

  • confidence — 엣지 신뢰도, [0.5, 1.0] 범위로 클램프
  • time_decay — exp(−Δ년/16), 하한 0.85. 동률일 때 최신순 정렬용
  • consistency — 충돌 엣지(conflict)면 ×0.5

intent_boost — 의도 정합 가산

의도와 fact_type이 맞으면 살짝 올려줍니다. 예를 들어 ownership_in 질문이면 shareholder ×1.10.

기본 점수(매칭 안 됨) = 0.70
